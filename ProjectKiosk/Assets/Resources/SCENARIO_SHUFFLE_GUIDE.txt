================================================================================
                    SCENARIO SHUFFLE SYSTEM - SETUP GUIDE
================================================================================

OVERVIEW:
---------
The Scenario Shuffle System allows you to randomize the order of customer
scenarios in the gameplay scene. It also supports "follow-up" scenarios that
play immediately after specific scenarios, regardless of shuffle order.

CURRENT SETUP:
--------------
✓ Fridgy scenario
✓ Shady Guy scenario (ShaunBakersScenario)
✓ Both scenarios are shuffled randomly

UNITY SETUP (DO THIS IN EDITOR):
---------------------------------
1. Open your Gameplay Scene
2. Find or create an empty GameObject (e.g., "GameplayManager")
3. Add the "ScenarioManager" component to it
4. Find the "DialogueManager" GameObject
5. In DialogueManager Inspector, drag the ScenarioManager GameObject into
   the "Scenario Manager" field
6. IMPORTANT: Clear the "Json Customer File Names" list in DialogueManager
   (This old system is deprecated - ScenarioManager handles it now)

HOW IT WORKS:
-------------
1. ScenarioConfig.json defines which scenarios exist and their order settings
2. ScenarioManager loads the config on start
3. If shuffle is enabled, it randomizes the scenario order
4. DialogueManager asks ScenarioManager for the next scenario
5. When a scenario completes, it checks if there's a follow-up scenario
6. Follow-up scenarios are inserted at the front of the queue

TESTING:
--------
- Run the gameplay scene
- You should see either Fridgy or Shady Guy appear first (random)
- After completing one, the other should appear
- Check the Unity Console for logs showing the shuffled order

ADDING THE POLICE SCENARIO (FUTURE):
-------------------------------------
When you're ready to add the Police scenario:

1. Create "PoliceScenario.json" in Assets/Resources/
2. Create the Police NPC prefab
3. Edit ScenarioConfig.json and change:

   FROM:
   {
     "filename": "ShaunBakersScenario",
     "displayName": "Shady Guy",
     "followUpScenario": ""
   }

   TO:
   {
     "filename": "ShaunBakersScenario",
     "displayName": "Shady Guy",
     "followUpScenario": "PoliceScenario"
   }

4. Add the Police scenario entry:
   {
     "filename": "PoliceScenario",
     "displayName": "Police Officer",
     "followUpScenario": ""
   }

RESULT:
- Fridgy and Shady will shuffle randomly
- Whenever Shady appears (first, middle, or last), Police will ALWAYS
  play immediately after
- Police is not added to the shuffle pool (it's triggered by Shady)

EXAMPLE SHUFFLE ORDERS:
-----------------------
Possible Run 1:
  1. Fridgy
  2. Shady
  3. Police (triggered by Shady)

Possible Run 2:
  1. Shady
  2. Police (triggered by Shady)
  3. Fridgy

TROUBLESHOOTING:
----------------
- If scenarios don't shuffle: Check "shuffleEnabled": true in ScenarioConfig.json
- If scenarios don't load: Check the Unity Console for error messages
- If DialogueManager not working: Make sure ScenarioManager reference is set
- If old behavior persists: Clear jsonCustomerFileNames list in DialogueManager
